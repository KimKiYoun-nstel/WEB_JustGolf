# Phase 5 구현 계획 (2026-02-09)

## 목표
- 일반 사용자 첫 로그인 이후의 네비게이션 경험을 개선한다.
- 참가 신청 흐름을 계정 승인 및 참가 여부에 맞게 분기한다.
- 본대회 및 사전/사후 라운드 UX를 단순화한다.
- 계정 기준 복수 참가자 등록과 관계 정보 저장을 지원한다.
- 라운드 관리자 권한 부여 방식을 계정 선택/검색 기반으로 전환한다.
- 대회별 경품 지원 기능을 추가한다.

## 구현 범위
1) /start 페이지 신설 + 더미 페이지 제공
- 일반 사용자 로그인 후 기본 랜딩을 /start로 변경
- /start에는 대회/제주달콧/게시판 바로가기 제공
- /jeju, /board 더미 페이지 추가 (레이아웃 + 홈 링크)

2) 대회 상세 진입 분기
- /t/[id] 접근 시 본인 참가 신청 여부 확인
- 이미 신청한 경우 참가자 현황 페이지로 이동
- 신청하지 않은 경우 참가 신청 폼 유지

3) 신청 폼에 “미정” 옵션 추가
- 식사, 카풀, 이동수단, 사전/사후 라운드 등 모든 항목에 미정 추가
- 본대회 신청 상태에도 “미정” 상태를 허용

4) 계정 기준 복수 참가자 신청
- 한 계정이 여러 참가자를 등록 가능
- 참가자별 관계 정보를 자유 입력으로 저장
- 참가자 목록 및 카운트 반영 방식 조정

5) 사전/사후 라운드 관리 단순화
- 상태는 신청/대기만 유지
- 취소된 항목은 목록에서 숨김
- 숙박/식사 참여 여부를 현황에 출력

6) 라운드 관리자 권한 부여 UX 개선
- UUID 입력 방식 제거
- 계정 가입자 리스트 + 검색(이메일/닉네임) 기반 선택

7) 경품 지원 페이지 추가
- 대회별 경품 지원 정보 입력/리스트 페이지 추가
- 대회 상세에 경품 지원 현황 섹션 노출

8) 관리자 대회 목록 버튼 정리
- 가입승인 버튼 제거 (계정 승인으로 대체됨)

## 검증 시나리오
- 미승인 사용자 /start 접근 차단 및 로그인 메시지 확인
- /t/[id] 진입 시 신청 여부에 따른 페이지 분기 확인
- 미정 옵션 저장/표시 정상 동작 확인
- 복수 참가자 등록 시 관계 정보 및 카운트 반영 확인
- 라운드 관리에서 취소 숨김 및 상태 단순화 확인
- 라운드 관리자 권한 부여 검색/선택 흐름 확인
- 경품 지원 입력/리스트 및 상세 표시 확인

## 비고
- DB 마이그레이션이 필요한 항목은 별도 파일로 분리하여 순차 적용
- 정책(RLS) 변경은 승인/권한 흐름과 충돌 없도록 검증 필요
