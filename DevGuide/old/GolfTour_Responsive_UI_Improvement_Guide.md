# Golf Tour — 모바일 반응형(UI) 개선 가이드

작성일: 2026-02-09  
목적: 현재 배포본(모바일)에서 보이는 **세로 글씨/컬럼 깨짐/버튼 난잡함**을 해결하고, 전반적인 **반응형 웹 품질**을 올리기 위한 작업 가이드.

---

## 0) 결론(문제 성격)
- 이슈는 Vercel 문제가 아니라 **프론트 레이아웃(Tailwind/CSS/컴포넌트 구조)** 문제다.
- 모바일 폭에서 공간이 부족할 때
  - 헤더 메뉴가 줄바꿈/축소가 이상하게 걸리며 “세로 글씨”처럼 보이고,
  - 관리자 페이지 테이블은 컬럼이 폭을 초과해 찌그러진다.

---

## 1) 현재 화면에서 확인된 구체적 개선 포인트(우선순위)

### 1-1. 상단 헤더: 메뉴/텍스트가 세로로 깨짐
**증상**
- `대회 목록`, `관리자` 같은 메뉴가 모바일 폭에서 줄바꿈이 이상하게 되거나 세로로 찌그러짐

**권장 해결(강추) — 모바일에서 헤더 단순화 + 햄버거 메뉴**
- 모바일(`md 미만`)에서는:
  - 로고 + (햄버거 버튼) + 프로필 정도만 노출
  - 메뉴(대회목록/관리자/내프로필)는 **드로어/시트(Sheet)** 로 이동
- 데스크탑(`md 이상`)에서는:
  - 기존처럼 메뉴를 상단에 펼쳐서 표시

**Tailwind 적용 패턴 예시(개념)**
- 데스크탑 메뉴: `hidden md:flex`
- 모바일 햄버거: `flex md:hidden`
- 상단 바: `flex items-center justify-between gap-2`

**추가 팁**
- 헤더 내부 텍스트는 `whitespace-nowrap`를 적절히 사용하고,
- 이메일 같은 긴 문자열은 `truncate max-w-[…]`로 잘라서 레이아웃을 보호

---

### 1-2. 관리자(/admin) 대회 목록: 테이블 컬럼/작업 버튼이 찌그러짐
**증상**
- 모바일에서 테이블(대회명/일정/상태/작업)이 폭을 초과하여 컬럼이 무너짐
- 작업 버튼(수정/가입승인/신청자/파일/라운드…)이 한 컬럼에 몰리며 세로로 난잡해짐

#### 해결안 A(추천) — 모바일은 “카드 리스트”, 데스크탑은 “테이블”
- 모바일: 카드 UI로 변환
  - 상단: 대회명
  - 아래: 일정/상태
  - 하단: 작업은 버튼 1~2개 + “더보기(…)" 드롭다운
- 데스크탑: 기존 테이블 유지

> UX가 가장 좋고, 모바일에서 특히 자연스럽다.

#### 해결안 B(빠른 임시처방) — 테이블 가로 스크롤 허용
- 테이블을 감싸는 래퍼에:
  - `overflow-x-auto`
  - `whitespace-nowrap`
- 테이블 자체는 폭이 넓어도 되고, 모바일에서 가로 스크롤로 본다.

> 가장 빠르게 “안 깨지게” 만들 수 있다. 다만 모바일 UX는 A보다 떨어질 수 있음.

#### “작업” 컬럼은 드롭다운으로 축약(강추)
- 모바일에서 버튼을 여러 개 나열하지 말고:
  - `작업` 버튼 하나(또는 `…`)를 누르면 메뉴가 뜨게 만들기
- shadcn/ui를 쓰고 있다면 `DropdownMenu`가 정답

## 2) 반응형 웹 전반 개선 체크리스트(일반 가이드)

### 2-1. 레이아웃 기본(컨테이너/패딩)
- 페이지 래퍼에 기본적으로 다음을 권장:
  - `container mx-auto px-4`
- 카드/섹션은 `space-y-4` 같은 간격을 일관되게 유지

### 2-2. 타이포그래피(글자 크기/줄바꿈)
- 모바일에서는 글자가 크면 줄바꿈 폭탄이 난다.
- 패턴:
  - 제목: `text-xl md:text-2xl`
  - 본문: `text-sm md:text-base`

### 2-3. 긴 텍스트(이메일/코스명/제목) 처리
- 긴 텍스트는 레이아웃을 파괴하는 주범
- 이메일/긴 제목:
  - `truncate` + `max-w-[200px]`(모바일) 같은 상한
  - 필요 시 툴팁/상세 페이지에서 전체 표시

### 2-4. 버튼/입력(모바일 터치 UX)
- 버튼은 모바일에서 **최소 높이 44px** 권장(터치 영역)
- 패턴:
  - 모바일: `w-full`
  - 데스크탑: `md:w-auto`
- 폼은 `gap-3` 이상으로 간격 확보

### 2-5. 테이블은 모바일에서 원래 취약하다
- 모바일에서 테이블은 보통 다음 중 하나로 푼다:
  1) 카드 리스트로 변환(추천)
  2) 가로 스크롤 허용(빠른 해결)
  3) 컬럼 숨김(모바일에선 일정/상태만, 작업은 드롭다운)

### 2-6. 반응형 브레이크포인트 운영 방식(권장)
- `sm`: 휴대폰 가로/큰 폰
- `md`: 태블릿/작은 노트북
- `lg`: 데스크탑
- “모바일 우선”으로 기본 스타일을 모바일에 맞추고, `md:` 이상에서 확장하는 방식이 편함

## 3) 구현 순서 추천(최소 노력으로 효과 크게)

### Step 1 — 관리자 대회 목록 ‘안 깨지게’ 만들기(즉효)
- (빠른 처방) 테이블 래퍼에 `overflow-x-auto` 적용
- (권장) 작업 컬럼을 드롭다운으로 축약

### Step 2 — 헤더를 모바일 친화적으로 단순화
- 모바일: 햄버거/드로어로 메뉴 이동
- 이메일/프로필 문자열 truncate 적용

### Step 3 — 카드/리스트 UI를 모바일 기준으로 다듬기
- 카드 내 버튼은 모바일에서 `w-full`
- 불필요한 텍스트 줄이고 핵심만 표시

### Step 4 — 폼/상세 페이지(신청/조편성 등) 모바일 최적화
- 입력 필드 간격/버튼 위치/스크롤 경험 개선

---

## 4) 작업 시 참고 팁
- 크롬 개발자도구 → 모바일 디바이스 모드에서 너비 360~430px로 반복 확인
- “텍스트가 레이아웃을 깨는지”를 먼저 의심 (truncate/nowrap)
- “표가 깨지는지”는 카드 변환 or 스크롤 허용 중 하나로 확실히 결정

---

## 5) 다음에 같이 잡으면 좋은 것(선택)
- `/admin` 뿐 아니라 `/admin/tournaments/{id}/edit` 같은 폼 화면도 모바일에서 더 빡빡할 수 있음
- `/t/{id}` 상세에서 신청/현황 섹션이 테이블이면 동일한 전략 적용

---

### 메모(파일/컴포넌트 확인 후 더 정확한 지시 가능)
헤더 컴포넌트 파일 경로(예: `components/header.tsx`)와  
관리자 대회 목록 컴포넌트 파일(예: `app/admin/tournaments/page.tsx`)만 알려주면,
- 정확히 어떤 Tailwind class를 어디에 바꿔야 하는지
- “모바일 카드 vs 데스크탑 테이블”을 실제 코드 구조에 맞춰
더 구체적으로 작업 지시서를 업데이트할 수 있음.
