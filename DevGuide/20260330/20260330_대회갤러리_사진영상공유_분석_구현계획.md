> 브랜치: `feature/tournament-gallery`

## 1. 목표 및 배경

종료된 대회(`status = 'done'`)에 한해, 참가자들이 대회 당일 찍은 **사진과 영상을 공유**할 수 있는 공간을 제공한다.  
컨셉: **대회 전용 인스타그램** — 업로드, 좋아요, 댓글, YouTube 링크 포함.

현재 결과 보기 페이지(`/t/[id]/results`)에서 **갤러리 링크를 추가**하고, 별도 페이지(`/t/[id]/gallery`)에서 기능을 제공한다.

## 2. 핵심 제약 조건: Supabase 스토리지

### 2-1. 현재 Supabase 플랜별 스토리지 한도

| 플랜 | 스토리지 | 파일 업로드 최대 크기 | 비용 |
|---|---|---|---|
| Free | 1 GB | 50 MB | 무료 |
| Pro | 100 GB | 50 MB | $25/월 + $0.021/GB 초과 |

### 2-2. 문제점 분석

- **이미지**: 스마트폰 원본 사진 = 보통 3~10 MB. Free 플랜 1GB 기준 100~300장 저장 후 포화.
- **영상**: 1분 영상 = 약 50~200 MB. 직접 업로드는 **실질적으로 불가능** (용량 감당 불가).
- **비용 리스크**: 대회마다 100~200장 업로드 시 누적 용량이 빠르게 증가.

### 2-3. 권장 전략: **Hybrid Storage**

```
이미지 ──► Supabase Storage (클라이언트 압축 후 업로드)
              - 업로드 전 브라우저에서 리사이즈/압축 (canvas API)
              - 최대 해상도: 1920×1080 이하
              - 최대 파일 크기: 2 MB (압축 후)
              - 원본 보존 없음 (압축만 허용)

영상   ──► 외부 링크만 허용 (직접 업로드 완전 차단)
              - YouTube URL
              - Vimeo URL
              - 링크에서 썸네일 자동 추출 (oEmbed)
```

이 전략으로 **대회당 약 200장 기준 약 400 MB 소비** → Free 플랜에서 2~3개 대회 운영 가능.  
장기적으로 Pro 플랜 전환 또는 Cloudinary 같은 이미지 CDN 연동 고려.

## 4. DB 스키마 설계 (Migration 049)

```sql
-- ============================================================================
-- migration: 049_tournament_gallery.sql
-- ============================================================================

-- 4-1. 갤러리 아이템 테이블
create table if not exists public.tournament_gallery_items (
  id            bigint generated by default as identity primary key,
  tournament_id bigint not null references public.tournaments(id) on delete cascade,
  user_id       uuid not null references auth.users(id) on delete cascade,

  -- 미디어 타입
  media_type    text not null check (media_type in ('image', 'video_link')),

  -- 이미지: Supabase Storage path (e.g. "tournament-gallery/68/uuid.jpg")
  storage_path  text,
  -- 이미지 공개 URL (저장 시점에 계산하여 저장)
  public_url    text,

  -- 영상 링크: YouTube/Vimeo URL
  video_url     text,
  -- 썸네일 URL (oEmbed 응답에서 추출하여 저장)
  thumbnail_url text,

  -- 공통
  caption       text,           -- 설명 (선택)
  is_hidden     boolean not null default false,  -- 관리자 숨김
  created_at    timestamptz not null default now(),
  updated_at    timestamptz not null default now(),

  -- 이미지 or 영상 링크 중 하나는 반드시 있어야 함
  constraint chk_media_content check (
    (media_type = 'image' and storage_path is not null)
    or
    (media_type = 'video_link' and video_url is not null)
  )
);

create index if not exists idx_gallery_items_tournament
  on public.tournament_gallery_items(tournament_id, created_at desc);

create index if not exists idx_gallery_items_user
  on public.tournament_gallery_items(user_id);

-- 4-2. 좋아요 테이블
create table if not exists public.tournament_gallery_likes (
  id      bigint generated by default as identity primary key,
  item_id bigint not null references public.tournament_gallery_items(id) on delete cascade,
  user_id uuid not null references auth.users(id) on delete cascade,
  created_at timestamptz not null default now(),

  unique (item_id, user_id)  -- 동일 사용자 중복 좋아요 불가
);

create index if not exists idx_gallery_likes_item
  on public.tournament_gallery_likes(item_id);

-- 4-3. 댓글 테이블
create table if not exists public.tournament_gallery_comments (
  id      bigint generated by default as identity primary key,
  item_id bigint not null references public.tournament_gallery_items(id) on delete cascade,
  user_id uuid not null references auth.users(id) on delete cascade,
  content text not null check (char_length(content) between 1 and 500),
  is_hidden boolean not null default false,
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now()
);

create index if not exists idx_gallery_comments_item
  on public.tournament_gallery_comments(item_id, created_at);
```

## 5. RLS 정책 설계

### tournament_gallery_items

| 작업 | 허용 대상 | 조건 |
|---|---|---|
| SELECT | 로그인 사용자 | `is_hidden = false` 또는 관리자 |
| INSERT | 승인된 로그인 사용자 | `is_approved_user(auth.uid()) = true` + 대회 `done` 상태 확인 |
| UPDATE | 본인(caption 수정) 또는 관리자(is_hidden) | `user_id = auth.uid()` OR `is_admin_secure` |
| DELETE | 본인 또는 관리자 | `user_id = auth.uid()` OR `is_admin_secure` |

> ⚠️ INSERT 시 대회 `status = 'done'` 확인을 서버(Route Handler)에서도 이중 체크

| SELECT | 로그인 사용자 |
| INSERT | 로그인 사용자 (본인 `user_id` 일치 필수) |
| DELETE | 본인만 (`user_id = auth.uid()`) |

## 6. Supabase Storage 버킷 설정

```
버킷명: tournament-gallery
접근: private (Signed URL 또는 Public URL — 아래 선택지 참고)
```

### 공개 방식 선택지

| 방식 | 장점 | 단점 |
|---|---|---|
| **Public 버킷** | URL 만으로 직접 접근, 구현 단순 | 버킷 전체 공개 (URL 알면 누구나 접근) |
| **Private + Signed URL** | 접근 제어 가능 | Signed URL 만료(기본 1시간) → 매번 갱신 필요 |
| **Private + Public URL** | 버킷 비공개이나 특정 폴더 공개 | 설정 복잡 |

> ✅ **권장**: `tournament-gallery` 버킷을 **Public**으로 설정, 업로드 경로를 `{tournament_id}/{uuid}.{ext}` 형태로 구조화.  
> 삭제 시 Storage에서도 함께 삭제 처리 (Route Handler 또는 DB trigger).

### 업로드 경로 규칙
```
tournament-gallery/{tournament_id}/{user_id}/{uuid}.jpg
```

### 클라이언트 이미지 압축 전략
- 라이브러리: `browser-image-compression` (npm) — 1줄 압축, Next.js App Router 호환
- 설정:
  ```js
  maxSizeMB: 2,
  maxWidthOrHeight: 1920,
  useWebWorker: true,
  ```

## 9. YouTube 링크 처리 방식

### oEmbed를 통한 썸네일 추출

```
https://www.youtube.com/oembed?url={video_url}&format=json
응답: { thumbnail_url, title, ... }
```

- 업로드 시 서버(Route Handler)에서 oEmbed API 호출
- `thumbnail_url`을 `tournament_gallery_items.thumbnail_url`에 저장
- 클라이언트에서는 저장된 thumbnail_url을 사용 (oEmbed 재호출 불필요)

### 지원 플랫폼
- YouTube: `youtube.com/watch?v=`, `youtu.be/`
- Vimeo: `vimeo.com/{id}`

## 10. 위험 요소 및 고려사항

| 항목 | 위험도 | 대응 |
|---|---|---|
| Supabase Storage 용량 초과 | ⚠️ 중간 | 클라이언트 압축(2MB 제한) + 대회별 업로드 한도(예: 인당 20장) 설정 |
| 부적절한 이미지 업로드 | ⚠️ 중간 | 관리자 숨김 기능(Phase 3), 신고 버튼(선택) |
| 영상 직접 업로드 요청 | 낮음 | 정책 고지: "영상은 YouTube 링크만 지원" |
| 비로그인 접근 차단 | 낮음 | 갤러리는 `로그인 + 승인` 사용자만 열람/업로드 |
| Supabase Storage 삭제 동기화 | ⚠️ 중간 | DB 삭제 시 Route Handler에서 Storage 파일도 함께 삭제 |
| 이미지 URL 영속성 | 낮음 | Public 버킷 사용 시 URL 영구 유효 |

## 11. 인당 업로드 한도 (권장)

Supabase Free 플랜 1GB 기준, 대회 1개당 운용 가능한 이미지 수:
- 압축 후 평균 1.5 MB 기준: **약 666장**
- 인당 20장 제한 시: **약 33명**이 업로드 가능
- 인당 10장 제한 시: **약 66명**이 업로드 가능

> ✅ 실제 대회 참가자가 20~30명 규모임을 감안, **인당 20장, 대회 전체 최대 500장(~750MB)** 제한 권장.
